### Sticky Positioning

- Class: `sticky top-0 z-10`
- Background must be opaque so grid does not bleed through on scroll: `bg-gray-50 dark:bg-gray-900` (matches page background; source: `page.tsx` `main` element)
- Bottom border: `border-b border-gray-200 dark:border-gray-700` — visual separation from content below (matches SearchBar's input border tone)
- Vertical padding: `py-2` (8px top + 8px bottom) → rendered bar height ≈ 40–48px including button height

### Scroll Offset (IMPL-02, Pitfall 9)

- Anchor wrappers in the grid use Tailwind class `scroll-mt-16` (64px)
- 64px provides sufficient clearance for the sticky `AlphabetBar` height (~48px) plus 16px breathing room
- Applied only to the **first** book `<div>` per letter group (`isFirstInGroup === true`)
- Class must be a full literal: `"scroll-mt-16"` — no dynamic construction (Tailwind v4 purge rule)

### Grid Anchor Wrappers

```tsx
// In filteredBooks.map:
<div
  key={book.id}
  className={isFirstInGroup ? "scroll-mt-16" : undefined}
  {...(isFirstInGroup ? { "data-letter": initial } : {})}
>
  <BookCard book={book} />
</div>
```

- The wrapper `<div>` has **no layout classes** — it is a transparent grid child
- Do NOT use `display: contents` (breaks `querySelector` and accessibility tree in some browsers; source: ARCHITECTURE.md)

## Spacing Scale

Declared values (multiples of 4, inheriting from existing 8-point scale in codebase):

| Token | Tailwind | Value | Usage |
|-------|----------|-------|-------|
| xs | `gap-1` | 4px | Gap between letter buttons on mobile |
| sm | `gap-2` / `px-2` / `py-1` | 8px | Letter button padding; gap between buttons desktop; bar internal gaps |
| md | `gap-3` / `px-3` | 12px | Sort toggle button horizontal padding; gap on sm+ breakpoint |
| lg | `mb-6` | 24px | Bottom margin below `AlphabetBar` before grid — **decision:** AlphabetBar uses `mb-6` (not `mb-8`) so it does not double-stack with SearchBar's existing `mb-8` above it; combined gap between search input and grid is 24px, not 64px |
| xl | `py-2` | 8px top+bottom | AlphabetBar bar padding |

> **Authoritative source:** The component spec table below (letter button sizes, padding, gap) is the single source of truth. Prose descriptions elsewhere in this spec (e.g. "approximately N px wide") are illustrative only and should not be used to infer spacing values.

**Touch target minimum:** Letter buttons must hit 32×32px minimum. At `text-xs px-2 py-1` each button is approximately 28×24px — add `min-w-[1.75rem] min-h-[2rem]` to ensure 44px is met on mobile where `text-xs` is used. Use `min-w-[1.75rem]` (28px) on desktop where `text-sm` naturally clears 32px.

Exceptions: `scroll-mt-16` (64px scroll margin) — intentional non-standard value to clear sticky bar height.

## Component Specifications

### AlphabetBar Props Interface

```typescript
interface AlphabetBarProps {
  letters: string[];            // active letters Set → Array, from availableLetters in page.tsx
  activeLetter?: string | null; // currently selected letter (null = none selected)
  sortMode: 'author' | 'title'; // for future aria-label context; bar renders same regardless
  onLetterClick: (letter: string) => void;
  onSortChange: (mode: 'author' | 'title') => void;
}
```

### Sort Toggle — Segmented Control

- Shape: two adjacent `<button>` elements inside a `<div>` with `rounded-lg border border-gray-200 dark:border-gray-700 overflow-hidden`
  - No gap between buttons — they share the container border; an inner divider `w-px bg-gray-200 dark:bg-gray-700` separates the two (this is a `<span>`, not a bordered element — use `bg-*` not `border-*`)
- Width: `shrink-0` — does not stretch; sits left of the letter run
- Each button: `px-3 py-1 text-xs sm:text-sm transition-colors`
- Selected state: `bg-blue-500 text-white font-semibold`
- Unselected state: `bg-white dark:bg-gray-800 text-gray-700 dark:text-gray-300 hover:bg-gray-100 dark:hover:bg-gray-800`

### Letter Buttons

- Element: `<button>` (not `<a>`, not `<div>`) — required for native `disabled` attribute and keyboard focus
- Size: `w-7 h-8 text-xs sm:text-sm font-medium rounded` — 28×32px base, sufficient for pointer targets; mobile wraps via `flex-wrap`
- Transition: `transition-colors duration-150` — no bounce or scale animation on letters (anti-feature; source: FEATURES.md)
- Disabled letter classes (full literal): `"text-gray-400 dark:text-gray-500 opacity-40 cursor-not-allowed"`
- Active/available letter classes (full literal): `"text-gray-700 dark:text-gray-300 hover:bg-gray-100 dark:hover:bg-gray-800 rounded"`
- Selected letter classes (full literal): `"bg-blue-500 text-white rounded"`
- Focus-visible classes (applied to all): `"focus-visible:outline-none focus-visible:ring-2 focus-visible:ring-blue-500 focus-visible:ring-offset-1"`

### Mobile Layout (IMPL-04)

- Container: `flex flex-wrap justify-start items-center gap-1 sm:gap-2`
- On 375px screen: `text-xs` + `px-2 py-1` → each button ≈ 28px wide → 26 buttons ≈ 728px natural width → wraps to ~2 rows
- Two-row wrap is acceptable (iOS Contacts precedent); bar height increases to ~72–80px when wrapped → update `scroll-mt` to `scroll-mt-24` **only if** actual rendered height exceeds 56px in testing
- **Default contract:** `scroll-mt-16` (64px); adjust to `scroll-mt-20` (80px) if testing reveals two-row wrap on 375px hides cards
- No horizontal overflow: `overflow-x-hidden` on the `<nav>` container

## Interaction States

### Letter Button State Machine

```
                    ┌─────────────────┐
                    │    DISABLED     │ ← isActive === false
                    │ opacity-40      │
                    │ cursor-not-     │
                    │   allowed       │
                    │ NOT focusable   │
                    └─────────────────┘

  ┌──────────┐  hover  ┌──────────────┐  click   ┌──────────────────┐
  │ INACTIVE │ ──────► │    HOVER     │ ────────► │    SELECTED      │
  │ (resting)│         │ bg-gray-100  │           │ bg-blue-500      │
  │          │ ◄────── │              │           │ text-white       │
  └──────────┘  leave  └──────────────┘           │ aria-current=true│
       ▲                                          └──────────────────┘
       │                                                   │
       └───────────── sortMode change ◄────────────────────┘
                      or searchTerm change (reset activeLetter)
```

### Sort Toggle State Machine

```
  [Author SELECTED] [Title unselected]
        │
        │ user clicks "Title"
        ▼
  [Author unselected] [Title SELECTED]
        + activeLetter resets to null
        + grid re-sorts immediately
        + availableLetters recomputes
```

### Framer Motion Guard (IMPL-02)

- `gridReady` state: `const [gridReady, setGridReady] = useState(false)`
- Set via: `onAnimationComplete={() => setGridReady(true)}` on the `motion.div` grid wrapper
- `handleLetterClick` noop guard: `if (!gridReady) return;`
- `AlphabetBar` receives: `disabled={!gridReady}` as a prop override OR letter buttons check `gridReady` via closure in `page.tsx`'s `handleLetterClick` — the bar itself is always rendered but clicks are intercepted at the handler level
- **Alternative**: Set `initial={false}` on `motion.div` for subsequent renders (after initial mount) so the animation only runs once at page load; `gridReady` still guards the 500ms window

## Accessibility Contract (IMPL-03)

### Semantic Structure

```tsx
<nav aria-label="Jump to books by letter">
  {/* Sort toggle */}
  <div role="group" aria-label="Sort order">
    <button
      aria-pressed={sortMode === 'author'}
      onClick={() => onSortChange('author')}
    >Author</button>
    <button
      aria-pressed={sortMode === 'title'}
      onClick={() => onSortChange('title')}
    >Title</button>
  </div>

  {/* Letter buttons */}
  {ALPHABET.map(letter => (
    <button
      key={letter}
      disabled={!letters.includes(letter)}
      aria-label={`Jump to books starting with ${letter}`}
      aria-current={activeLetter === letter ? true : undefined}
      onClick={() => onLetterClick(letter)}
    >
      {letter}
    </button>
  ))}
</nav>
```

### ARIA Rules

| Element | Required Attribute | Value |
|---------|--------------------|-------|
| `<nav>` container | `aria-label` | `"Jump to books by letter"` |
| Sort toggle group | `role="group"` + `aria-label` | `"Sort order"` |
| Sort toggle button | `aria-pressed` | `true` when selected, `false` otherwise |
| Active/available letter | `aria-label` | `"Jump to books starting with {letter}"` |
| Selected letter | `aria-current` | `true` (not `"page"` — this is not page navigation) |
| Inactive letter | `disabled` | native HTML attribute — not `aria-disabled` |

**Do NOT use `aria-disabled`** as a substitute for `disabled` — `aria-disabled` keeps the element focusable and requires manual keyboard interception (source: PITFALLS.md, Pitfall 7).

### Keyboard Navigation

| Key | Behavior |
|-----|----------|
| `Tab` | Moves focus through all enabled letter buttons and sort toggle buttons; disabled letters are skipped natively |
| `Enter` / `Space` | Activates focused button (scroll or sort change) |
| Arrow keys | Not required — the bar is not a `listbox` or `radiogroup`; standard tab navigation is sufficient for WCAG 2.1 AA |

### Screen Reader Announcements

- Screen reader will announce: *"Jump to books by letter, navigation"* on focus entering `<nav>`
- Active letter: *"Jump to books starting with M, current"* (via `aria-current`)
- Disabled letter: *"Jump to books starting with X, dimmed"* (native `disabled` prevents focus, so disabled letters are not announced during Tab navigation)

## Framer Motion Integration Note

### Context (source: `page.tsx`, PITFALLS.md)

The existing grid is wrapped in:
```tsx
<motion.div
  initial={{ opacity: 0, y: 20 }}
  animate={{ opacity: 1, y: 0 }}
  transition={{ duration: 0.5 }}
  className="grid grid-cols-2 sm:grid-cols-3 md:grid-cols-4 lg:grid-cols-5 gap-8"
>
```

The 500ms `y: 20 → 0` entry animation creates a scroll race condition — `scrollIntoView` fired during this window lands ~20px short.

### Guard Implementation

**Chosen approach: `onAnimationComplete` callback** (preferred over `initial={false}` so the entry animation is preserved for the initial page load experience):

```tsx
// In page.tsx state block
const [gridReady, setGridReady] = useState(false);

// On motion.div
<motion.div
  initial={{ opacity: 0, y: 20 }}
  animate={{ opacity: 1, y: 0 }}
  transition={{ duration: 0.5 }}
  onAnimationComplete={() => setGridReady(true)}
  className="grid ..."
>

// In handleLetterClick
const handleLetterClick = (letter: string) => {
  if (!gridReady) return;
  const target = document.querySelector(`[data-letter="${letter}"]`);
  target?.scrollIntoView({ behavior: 'smooth', block: 'start' });
};
```

**`scroll-margin-top` value:** `scroll-mt-16` (64px). This is the CSS-native scroll offset on the `[data-letter]` wrapper `<div>` — no `window.scrollBy` hack needed. Adjust to `scroll-mt-20` (80px) only if bar height exceeds 56px on mobile two-row wrap (verify in browser).
